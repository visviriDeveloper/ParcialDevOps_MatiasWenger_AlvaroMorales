# Guía de contribución Repositorio Evaluacion Parcial Dev Ops

Este repositorio esta pensado para que mejoremos nuestro manejo de git y GitHub

---

## Flujo de Trabajo (Git Flow)

1. No subir cambios directamente a `main`.
2. La rama de trabajo para integracion se llama `develop/`: Debes agregarle un nombre descriptivo con los siguientes prefijos:
   - `feature` para nuevas funcionalidades.
   - `fix` para corregir errores.
   - `docs` para documentación.
   - `refactor` para mejoras de código.
